# 部署上线

# 基于条件

  • 文档放置在项目的 docs 目录中；
  • 使用的是默认的构建输出位置；
  • VuePress 以本地依赖的形式被安装到你的项目中，并且配置了如下 npm scripts:
{
  "scripts": {
    "build": "vuepress build docs"
  }
}

# GitHub Pages

  1. 首先需要在 .vuepress/config.js 中设置正确的base：

    module.exports = {
    // 如果发布的地址为 https://<USERNAME>.github.io/<REPO>/，则 base 设置为 /REPO/
    base: '/REPO/'
}

    提示

    如果发布的地址是 https://<USERNAME>.github.io/，则可以省略 base 设置这一步。

  2. 然后在项目的更目录下创建 deploy.sh 文件，并写入以下命令

    #!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run build

# 进入生成的文件夹
cd docs/.vuepress/dist

# 如果是发布到自定义域名
# echo 'www.example.com' > CNAME

git init
git add -A
git commit -m 'deploy'

# 如果发布到 https://<USERNAME>.github.io
# git push -f git@github.com:<USERNAME>/<USERNAME>.github.io.git master

# 如果发布到 https://<USERNAME>.github.io/<REPO>
# git push -f git@github.com:<USERNAME>/<REPO>.git master:gh-pages

cd -

  3. 最后在 package.json 中写入以下 script

    "deploy": "bash deploy.sh"

    写入后，每次执行以下代码即可将文档部署到网站

    npm run deploy
